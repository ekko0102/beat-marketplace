# 開發計畫

**專案：** BeatMarketplace
**總預計工時：** 4 個階段

---

## Phase 1：基礎架構 + 製作人頁面

### 目標
建立整個系統的骨架，完成製作人相關功能。

### 任務清單
- [ ] 初始化 Next.js 前端專案
- [ ] 初始化 Express 後端專案
- [ ] 設定 docker-compose（PostgreSQL）
- [ ] 建立資料庫 Schema（所有資料表）
- [ ] 製作人 API（CRUD）
- [ ] JWT 認證系統（register / login）
- [ ] 製作人個人頁前端
- [ ] 製作人列表頁前端
- [ ] 圖片上傳（大頭照、Banner）
- [ ] Navbar + Footer 元件
- [ ] 全站暗色主題設定

### 完成條件
可以在網站上看到製作人列表，點進去看個人資料，製作人可以登入並編輯自己的資料。

---

## Phase 2：Beat 上傳 + 播放器

### 目標
製作人能上傳作品，訪客能試聽。

### 任務清單
- [ ] Beat 上傳 API（含音訊檔案處理）
- [ ] ffmpeg 音訊截取（試聽版）
- [ ] 伴奏市集頁面前端
- [ ] Beat 卡片元件
- [ ] 授權方案設定介面
- [ ] 底部懸浮播放器（Howler.js）
- [ ] 播放次數統計
- [ ] 製作人後台：上傳管理介面
- [ ] Beat 詳細頁

### 完成條件
訪客可以瀏覽伴奏、試聽，製作人可以上傳作品並設定授權方案。

---

## Phase 3：搜尋篩選 + 購物車 + 付款

### 目標
核心交易功能完成。

### 任務清單
- [ ] 搜尋 API（關鍵字 + 全文搜尋）
- [ ] 篩選 API（genre / mood / bpm / key / price）
- [ ] 篩選側欄 UI 元件
- [ ] 購物車狀態管理（zustand）
- [ ] 購物車頁面
- [ ] 結帳頁面
- [ ] 綠界 ECPay 金流串接
- [ ] 付款回調處理
- [ ] 安全下載 token 機制
- [ ] 付款成功頁面 + 下載連結
- [ ] Email 通知（買家 + 製作人）
- [ ] 製作人後台：訂單查看

### 完成條件
完整購買流程跑通：搜尋 → 試聽 → 加入購物車 → 付款 → 下載。

---

## Phase 4：聯絡系統 + 優化 + 部署

### 目標
補齊剩餘功能，優化體驗。

### 任務清單
- [ ] 聯絡製作人表單 + API
- [ ] 製作人後台：查看訊息
- [ ] RWD 行動版優化
- [ ] SEO（meta tags、sitemap）
- [ ] 效能優化（圖片 lazy load、API cache）
- [ ] 錯誤處理統一
- [ ] 基本 E2E 測試（Playwright）
- [ ] 正式環境部署設定

### 完成條件
所有功能完整，手機版正常，可對外開放。

---

## 技術債 / 未來版本

以下功能建議 v2 再做：

- 版稅分潤系統（Stripe Connect）
- 製作人收益儀表板
- 推薦演算法（根據瀏覽推薦相似 beats）
- 多語言（繁中 / 英文）
- Exclusive 授權拍賣功能
- 製作人認證徽章
- 雲端儲存遷移（AWS S3 / Cloudflare R2）
